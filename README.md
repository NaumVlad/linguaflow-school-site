# LinguaFlow — управление английской школой

Рабочий MVP на русском языке: календарь, ученики, группы/пары и абонементы. Проект не требует сборки и готов для GitHub Pages или Vercel.

## Возможности

- недельный и месячный календарь;
- создание, редактирование, удаление и перенос занятий перетаскиванием;
- ученики, преподаватели, группы и пары;
- отдельное время занятия для каждого дня ученика, например Пн 16:00 и Чт 18:30;
- понимание сокращений дней в AI-командах: ПН, ВТ, СР, ЧТ, ПТ, СБ и ВС в любом регистре;
- добавление, переименование и удаление преподавателей;
- индивидуальная посещаемость для учеников, пар и групп;
- абонементы на 1–12 уроков, автоматическое списание только после проведённого и посещённого урока;
- автоматический перенос номера урока при отмене или отсутствии ученика;
- встроенный чат-помощник: пропуски, отмены, проведение уроков, преподаватели и массовое расписание командами на русском языке;
- ручная корректировка остатка с историей изменений;
- фильтры по преподавателю, статусу и ученику;
- локальное хранение в IndexedDB, резервная копия JSON;
- экспорт расписания в ICS для Apple Calendar, Google Calendar и Outlook;
- адаптивный интерфейс для телефона и компьютера.

## Запуск

Это статический сайт. Откройте `index.html` через локальный веб-сервер, например:

```bash
npx serve .
```

Затем откройте адрес, показанный в терминале. При первом запуске загружаются демонстрационные записи.

## Публикация на GitHub Pages

1. Создайте пустой репозиторий на GitHub, например `linguaflow-school`.
2. В папке проекта выполните:

```bash
git init
git add index.html styles.css app.js README.md vercel.json supabase
git commit -m "Create LinguaFlow school scheduler"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/linguaflow-school.git
git push -u origin main
```

3. В GitHub откройте **Settings → Pages → Build and deployment**, выберите **Deploy from a branch**, ветку `main` и папку `/ (root)`.

Сайт появится по адресу `https://ВАШ_ЛОГИН.github.io/linguaflow-school/`.

## Публикация на Vercel

Импортируйте GitHub-репозиторий в Vercel и нажмите **Deploy**. Фреймворк указывать не нужно: проект статический. Файл `vercel.json` уже содержит правила маршрутизации и заголовки.

## Подключение Supabase для многопользовательской работы

Текущий MVP хранит данные только на одном устройстве. Для реальной школы и одновременной работы нескольких сотрудников:

1. Создайте проект на [Supabase](https://supabase.com/).
2. Откройте **SQL Editor** и выполните `supabase/schema.sql`.
3. В настройках проекта скопируйте `Project URL` и публичный `anon key`.
4. Подключите `@supabase/supabase-js` и вынесите операции `load()`/`save()` из `app.js` в отдельный репозиторий данных.
5. Включите Supabase Auth и скорректируйте RLS-политики из схемы под роли администратора и преподавателя. Никогда не добавляйте `service_role` key во фронтенд.

Схема уже моделирует учеников, группы, участников групп, занятия, абонементы и историю операций. Это позволяет заменить IndexedDB без изменения интерфейса.

## AI-помощник

В командах можно указывать сокращения дней `ПН`, `ВТ`, `СР`, `ЧТ`, `ПТ`, `СБ`, `ВС` и отдельное время для каждого дня. Поддерживаются оба формата времени: `16:30` и `16.30`. Например: `Добавь ученика Борис Бобков ПН 16.30 ПТ 19.30`.

Команды `Очисти базу учеников` и `Удали всех учеников` удаляют учеников, группы/пары и связанные занятия только после отдельного подтверждения. Команды `Отмени последнее действие` и `Отмени все предыдущие действия` восстанавливают изменения, выполненные AI-помощником после установки этой версии; хранится до 50 точек отмены.

Расписание для всех учеников можно создавать как на текущий, так и на следующий месяц: `Составь расписание для всех учеников на этот месяц` или `Создай расписание на следующий месяц для всех учеников`. Вариант `на тот месяц` также распознаётся как текущий месяц. Повторная команда учитывает уже существующие занятия и не создаёт дубликаты.

Команды `Добавь расписание для всех учеников до конца месяца` и `Создай расписание для всех учеников до конца месяца` создают занятия только от сегодняшней даты до последнего дня текущего месяца. Уже существующие занятия учитываются, дубликаты не создаются.

Точку отсчёта абонемента можно задать обычной фразой: `Измени 23.07 Даша Северин 1 из 8`, `23.07 у Даши Северин будет первый из восьми` или `Начни абонемент Даши Северин на 8 уроков с 23.07`. Указанное занятие получает заданный номер, а следующие запланированные занятия нумеруются автоматически; отменённые уроки и индивидуальные пропуски номер не занимают. Если занятия на стартовую дату ещё нет, помощник создаёт его по времени из карточки ученика.

В демо помощник выполняет локальные команды без передачи персональных данных наружу. Через чат доступны создание, редактирование и удаление учеников, преподавателей, групп и занятий, управление составом групп, посещаемостью и абонементами, перенос уроков и автоматическое создание до 8 занятий на следующий месяц. Для разных дней можно указывать разное время: «измени ученика Анна Петрова: понедельник 16:00, четверг 18:30». Последнее массовое создание можно отменить командой «Отмени последнее действие». Удаление ученика, преподавателя, группы или занятия требует отдельной команды «Подтверждаю».

Для понимания совершенно свободных формулировок в продакшене подключите серверный endpoint к OpenAI Responses API и разрешите модели вызывать только проверенные операции приложения: `find_student`, `update_attendance`, `update_lesson_status`, `create_month_schedule` и `undo_last_action`. API-ключ должен храниться только в переменных окружения серверной функции; его нельзя помещать в `app.js` или другие файлы браузера. Перед массовыми удалениями сервер должен требовать подтверждение пользователя.

## iCloud Calendar

Приватный календарь iCloud нельзя безопасно встроить на публичную страницу без авторизации пользователя. В MVP есть экспорт ICS. Для двусторонней синхронизации нужен защищённый серверный слой, который работает с CalDAV и хранит app-specific password только на сервере (например, Vercel Function + зашифрованные переменные окружения). Не размещайте пароль Apple ID или app-specific password в клиентском JavaScript.

## Данные и безопасность

- Демо-данные остаются в браузере пользователя.
- JSON-экспорт позволяет сделать резервную копию.
- Перед использованием с реальными детьми настройте авторизацию, RLS и регламент доступа к персональным данным.
